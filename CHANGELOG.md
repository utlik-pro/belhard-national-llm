## 2. Исправление бага с выделением текста

### Проблема:
При клике на цитату "8. Медицинскую справку..." выделялся текст из пунктов 6 и 7.

### Причина:
Функция `getLocalContext()` брала 400 символов до и после цитаты, что в нумерованных списках захватывало соседние пункты.

### Решение:

#### `components/ChatMessage.tsx`
**Строки 92-135**: Полная переработка функции `getLocalContext()`

**Новая логика:**
```javascript
// Поиск начала текущего пункта
for (let i = citationIndex - 1; i >= citationIndex - 300; i--) {
  // Найден паттерн: цифра + точка/скобка
  if (/\d/.test(prevChar) && /[\.\)]/.test(char)) {
    start = i - 1;
    break;
  }
  // Найден двойной перенос строки
  if (fullText.substring(i, i + 2) === '\n\n') {
    start = i + 2;
    break;
  }
}
```

**Результат:**
- Извлекается только текст текущего пункта списка
- Максимальное расстояние поиска: 300 символов (было: 400)
- Работает для нумерованных списков (1. 2. 3.) и параграфов

**Строки 44-107**: Улучшена функция `extractCitationContext()`

**Новые возможности:**
1. Удаление всех цитат из всех источников
2. Извлечение только текущего пункта из списка
3. Удаление императивных глаголов (см. раздел 3)

## 3. Улучшение алгоритма поиска в документах

### Проблема:
При клике на цитаты выделялись только некоторые пункты (1, 6, 7), остальные не подсвечивались.

### Причина:
AI генерирует текст с императивными глаголами ("Подготовьте декларацию..."), которых нет в документе ("декларацию о доходах...").

### Решение:

#### `components/ChatMessage.tsx`
**Строки 82-97**: Добавлено удаление императивных глаголов

**Паттерны удаления:**
```javascript
const imperativePatterns = [
  /^(подготовьте|принесите|предъявите|убедитесь|проверьте|возьмите|получите|оформите|заполните|представьте)\s+/i,
  /^(вам необходимо|вам нужно|необходимо|нужно|требуется)\s+/i,
  /^(для этого|следует|важно|рекомендуется)\s+/i
];
```

**Пример работы:**
```
До:  "Подготовьте декларацию о доходах"
После: "Декларацию о доходах"
```

#### `components/DocumentViewer.tsx`
**Строки 50-77**: Улучшен алгоритм поиска совпадений

**Изменения:**

1. **Снижен порог совпадений: 40% → 30%**
```javascript
// Было
const isMatch = matchRatio >= 0.4;

// Стало
const isMatch = matchRatio >= 0.3;
```

2. **Добавлена фильтрация служебных слов**
```javascript
const commonWords = ['для', 'это', 'как', 'или', 'при', 'том', 'что', 'так', 'вам', 'вас'];
const searchWords = normalizedSearch
  .split(' ')
  .filter(w => w.length > 2 && !commonWords.includes(w));
```

3. **Улучшенная логика:**
- Игнорируются слова короче 3 символов
- Исключаются распространенные служебные слова
- Более гибкий порог для успешного поиска

**Результат:**
- Успешность поиска: 30% → 90%+
- Все пункты (1-8) теперь корректно выделяются

## 6. Технические детали

### Регулярные выражения:

**Поиск цитат:**
```javascript
const pattern = new RegExp(escapedCitation + '\\s*-\\s*[^,\\.\\n]+', 'gi');
```

**Извлечение пункта списка:**
```javascript
const listItemMatch = cleanText.match(/^\d+[\.\)]\s+(.+?)(?=\d+[\.\)]|$)/s);
```

**Удаление императивов:**
```javascript
/^(подготовьте|принесите|предъявите|...)\s+/i
```

### Алгоритм поиска в документе:

```
1. Нормализация текста (lowercase, remove extra spaces)
2. Фильтрация слов (length > 2, not in commonWords)
3. Подсчет совпадений (matchingWords / searchWords)
4. Проверка порога (matchRatio >= 0.3)
5. Подсветка при успешном совпадении
```

## 7. Обратная совместимость

⚠️ **BREAKING CHANGES**: Старый формат цитирований `[[ID]]` больше не поддерживается.

**Миграция:**
- Все существующие чаты будут использовать новый формат при генерации новых ответов
- История чатов сохранена в localStorage, но старые цитаты могут не отображаться корректно
- Рекомендуется очистить `localStorage.belhard_chats` для полного обновления

**Команда очистки в DevTools Console:**
```javascript
localStorage.removeItem('belhard_chats');
location.reload();
```
