# 🧪 Программа тестирования RAG системы Belhard AI

## 🚀 Быстрый старт

### 1. Запуск приложения
```bash
npm install
npm run dev
```
Откройте браузер: **http://localhost:3000/**

### 2. Вход в систему
- Введите любой email (mock authentication)
- Система автоматически авторизует вас

### 3. Выбор департамента
- Нажмите на кнопку департамента в верхнем левом углу
- Выберите нужный отдел для тестирования

## ✅ Критерии успешного прохождения теста

### 🎯 Обязательные проверки:

1. **Цитирование источников**
   - [ ] В ответе присутствуют ссылки вида [[ID]]
   - [ ] Цитаты соответствуют реальному содержанию документов
   - [ ] Источники цитируются сразу после утверждения

2. **Точность данных**
   - [ ] Цифры (ставки, проценты, сроки) совпадают с документами
   - [ ] Указываются конкретные статьи и пункты
   - [ ] Нет противоречий с законодательством

3. **Структурированность ответа**
   - [ ] Ответ представлен в виде пошаговой инструкции (где применимо)
   - [ ] Используется markdown форматирование
   - [ ] Информация логически организована

4. **Отображение источников в UI**

## 🔧 Troubleshooting

### Проблема: Нет цитирования источников

**Симптомы:** AI отвечает, но не ставит [[ID]] в тексте

**Решение:**
1. Проверьте `systemInstruction` в `services/mockApiService.ts` (строки 142-166)
2. Убедитесь, что инструкция требует формат `[[ID]]`
3. Проверьте, что `contextSources` передаются в API (строка 121)

---

### Проблема: Неточные или устаревшие данные

**Симптомы:** AI называет неправильные цифры, сроки или ставки

**Решение:**
1. Откройте `constants.ts`
2. Найдите нужный источник по ID
3. Проверьте `fullContent` - возможно, данные нужно обновить
4. Убедитесь, что изменения сохранены

---

### Проблема: Источники не отображаются в UI

**Симптомы:** Ответ есть, но список источников внизу пустой

**Решение:**
1. Проверьте функцию `extractUsedSources()` в `services/mockApiService.ts` (строки 98-106)
2. Убедитесь, что regex `/\[\[(.*?)\]\]/g` корректно работает
3. Проверьте, что `onComplete(usedSources)` вызывается (строка 244)

---

### Проблема: API ошибки или нет ответа

**Симптомы:** "[Ошибка системы: Сервис временно недоступен]"

**Решение:**
1. Проверьте `.env.local` - установлен ли `GEMINI_API_KEY`
2. Проверьте консоль браузера на ошибки
3. Проверьте консоль dev-сервера на ошибки API
4. Убедитесь, что API ключ валидный и имеет квоту

---

### Проблема: Модальное окно не открывается

**Симптомы:** Клик по источнику не вызывает модальное окно

**Решение:**
1. Проверьте `components/ChatMessage.tsx`
2. Убедитесь, что `onViewSource` передается корректно
3. Проверьте `components/DocumentViewer.tsx`
4. Откройте консоль браузера для проверки ошибок

---

### Проблема: Slow performance / долгая загрузка

**Симптомы:** Ответы генерируются очень медленно

**Решение:**
1. Проверьте количество отправляемых источников (строка 121 в `mockApiService.ts`)
2. Уменьшите лимит с 20 до 10: `const contextSources = currentSources.slice(0, 10);`
3. Проверьте скорость интернета
4. Рассмотрите использование модели `gemini-2.0-flash` вместо `gemini-2.5-flash`
